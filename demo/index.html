Simulation Console — drive every use case live

Each control on the left triggers a real request against the running services through the same-origin nginx proxy. Every response slides in from the right; reopen the most recent one anytime with Last response in the header. The dashboard refreshes from the analytics service's own /v1/stats endpoints.

FR1 · Ingestion

Ingest events

POST /api/gateway/v1/events/ingest — a valid single event, a batch of 10, or a deliberately invalid event to prove validation.

FR5 + FR2 · Attack wave & scoring

Simulate an attack wave

Generates events in the browser and POSTs them in batches. Repeated (IP, path) bursts hit /admin//login to trip sensitive-path and repeat-offender scoring, mixed with varied categories, severities and actions.

FR2 · Scoring rules

Edit the rule engine

The enrichment service scores every event against these data-driven rules (/api/enrichment/v1/rules). Add, edit or delete a rule and it takes effect on the next event — no restart. Then run an attack wave to see scores shift.

FR1 + FR2 · Custom event

Craft & score one event

Set the fields a scoring rule keys on, send a single POST /v1/events/ingest, and read back the threat score the engine gave it — the quickest way to prove a rule you just added.

FR3 · Statistics

Refresh statistics

GET /api/analytics/v1/stats/summary — updates the KPI tiles, category / action bars, top attackers and top paths.

FR4 · Samples

Fetch event samples

GET /api/analytics/v1/events/samples — filtered, newest first.

B3 · Time-series

Event volume over time

GET /api/analytics/v1/stats/timeseries?interval= — pick a bucket size.

B1 · Alerting

Threshold alert rules

Define a rule (POST /alerts/define) then evaluate all rules (GET /alerts/evaluate).

B4 · Rate limiting

Per-IP rate limit

Fires 120 rapid GET /stats/summary requests and tallies 200 vs 429 (default limit ~100/min per client IP).

Note: through the proxy the console shares one client identity, so any other calls you make also count toward the same limit.

B2 · Streaming ingestion

Kafka consumer path

The gateway also consumes the Kafka events.ingest topic, so producers can stream events without HTTP. Browsers can't produce to Kafka — run this on the host to exercise that path:

./scripts/produce-to-kafka.sh 50
